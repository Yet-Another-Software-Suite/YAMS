Detailed Description

Configuration for the EasyCRT solver.

Required setup
Call WithTeeth() to supply the two encoder gear tooth counts and the common scale factor K. enc1Teeth and enc2Teeth must be coprime; the mechanism range must not exceed one CRT period (= lcm(enc1Teeth, enc2Teeth) / commonK rotations).
Offsets and inversion
If the encoder hardware supports on-device zeroing, leave the software offsets at zero to avoid double-offsetting. Use inv1 / inv2 only when the encoder physically reads backwards and cannot be inverted in firmware.

Set the maximum acceptable combined encoder error.

Each encoder's modular error is measured as a normalised fraction of one encoder rotation [0, 0.5). A combined error above tol causes the solver to report NoSolution. Tune against GetLastError() on your robot.

Parameters
